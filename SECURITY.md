# Security Policy

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

If you discover a security vulnerability in this project, please report it by emailing [support@sixtyfour.ai](mailto:support@sixtyfour.ai). Include as much detail as possible:

- A description of the vulnerability and its potential impact
- Steps to reproduce or a proof-of-concept
- Any suggested mitigations, if known

We will acknowledge your report within 3 business days and aim to provide a resolution timeline within 10 business days. We ask that you give us reasonable time to address the issue before any public disclosure.

## Scope

This policy applies to code maintained in this repository. If you believe you've found a vulnerability in the Sixtyfour platform itself, please report it through the same email address.

## Supported Versions

Only the latest release of this project receives security updates. We encourage all users to stay on the most recent version.

## Out of Scope

The following are generally not considered security vulnerabilities for this project:

- Issues in third-party dependencies (please report those to the respective upstream projects)
- Vulnerabilities requiring physical access to a user's device
- Social engineering attacks

Thank you for helping keep Sixtyfour projects and their users safe.
